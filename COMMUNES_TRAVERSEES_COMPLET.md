## Méthodologie

### Données sources
- **Tracés GPS** : 32 fichiers GPX du Chemin Huguenot (~52'000 points GPS)
- **Limites communales** : swissBOUNDARIES3D (version 2025-04, système LV95)

### Processus d'analyse
1. Fusion de tous les tracés GPX en une seule couche
2. Reprojection en coordonnées suisses LV95 (EPSG:2056)
3. Création d'un tampon de 300 mètres autour du tracé
4. Intersection spatiale avec les communes suisses
5. Extraction et nettoyage des données (nom, canton, numéro BFS)
6. Suppression des doublons

### Outils utilisés
- Python avec GeoPandas
- Script : `analyser_communes.py`

## Fichiers générés

- **`COMMUNES_TRAVERSEES_COMPLET.md`** : Ce document
- **`liste_communes_traversees.csv`** : Liste au format CSV (Excel)
- **`communes_traversees_complet.gpkg`** : Données géographiques complètes
- **`analyser_communes.py`** : Script d'analyse Python

## Notes

- Les entités "Bielersee", "Lac de Neuchâtel" sont des surfaces d'eau traversées
- Certaines communes apparaissent avec des précisions géographiques (ex: "Hausen (AG)", "Lohn (SH)") pour éviter les confusions
- Le tampon de 300m permet de capturer les communes directement traversées ainsi que quelques communes voisines très proches du tracé

## Communes bourgeoises

**Note** : Cette analyse identifie les **communes politiques** suisses. L'identification des **communes bourgeoises** (Bürgergemeinden) nécessiterait une analyse complémentaire avec les registres cantonaux spécifiques, car :
1. Toutes les communes n'ont pas de commune bourgeoise
2. Les limites des communes bourgeoises ne correspondent pas toujours aux limites politiques
3. Les données swissBOUNDARIES3D ne contiennent pas cette distinction

Pour obtenir la liste des communes bourgeoises, il faudrait consulter les registres cantonaux individuels.
